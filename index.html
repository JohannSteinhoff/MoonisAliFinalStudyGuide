1 · Core Definitions
AI (Traditional / Predictive)
The broad discipline of making systems that perform human-like tasks — reasoning, perception, learning, decision-making, problem-solving. Uses an algorithmic approach and produces accurate, consistent output. Analyzes data and makes predictions within a defined problem space. Key shortcoming: cannot create new, novel content.
Generative AI
A subset of AI focused on creating new content — text, images, audio, video, code, or synthetic data. Models underlying data distributions and samples from them. NOT algorithmic; can be inaccurate/inconsistent. Resolves AI's inability to create novel content.
💡
Generative AI doesn't replace traditional AI — it extends it into content creation. Real enterprise systems use both together.

1 · The Basic Unit: Neuron / Perceptron
🧠
A neural network is made of many neurons (perceptrons) stacked in layers. Each neuron takes inputs, does a calculation, and passes an output forward.

Bias — "The Starting Point"

  • A built-in offset that lets a neuron shift its output up or down
  • Allows the neuron to respond even when inputs are small
  • Gives the neuron a natural "starting tendency" independent of inputs
  • Without bias: rigid behavior. With bias: flexibility.

Activation Function — "The Decision Rule"

  • Decides how strongly the neuron responds after examining its inputs
  • Without it, the network acts like a simple calculator — no complexity
  • Introduces decisions, thresholds, on/off or gradual responses
  • Analogy: a light dimmer — small input = off, large input = bright
  • Common types: ReLU, sigmoid, tanh

Loss Function — "How Wrong Are We?"

  • Measures how bad a prediction is
  • Converts prediction quality into a single feedback signal
  • Analogy: a golf score — tells you how far off you were
  • Classification → "Was the label correct?"
  • Regression → "How far was the estimate from reality?"

Gradient — "How to Fix the Mistake"

  • Tells the model which direction to change its settings to reduce error
  • Answers: "Should this weight increase or decrease?"
  • Analogy: walking downhill in fog — you can't see the whole landscape but you feel which way slopes down
  • Takes small steps toward the lowest point (lowest error)
One-sentence summary: A neuron uses bias to set its default, activation to decide how strongly to respond, loss to measure how wrong it was, and gradients to learn how to do better — repeated millions of times.
2 · Neural Network Architecture

Input Layer

  • One neuron per input feature
  • No learned transformations — just passes data forward
  • Holds raw features (numbers, pixels, signals)

Hidden Layers

  • Fully connected to the previous layer (dense)
  • Each layer learns increasingly abstract representations
  • Number of neurons is a design choice (64, 128, 512…)
  • Nonlinear transformations happen here

Output Layer

  • Produces the final prediction
  • Number of neurons depends on the task
  • Activation function chosen based on task type

Weights & Biases

  • Learned parameters during training
  • Weights = strength between connected neurons
  • Biases = activation threshold adjustments
Training Workflow Data → Input Layer → Hidden Layers → Output Layer → Prediction → Compare with target → Backpropagate → Update parameters
📐
Network power comes from two things: depth (number of layers) and width (neurons per layer). Each layer does two things: linear transformation (mix & weight info) then nonlinear activation (bend the rules). Stack these enough times and you can approximate any complex function.
3 · Feed-Forward vs Backpropagation

Feed-Forward — "Making a Prediction"

  • Data enters at the input layer
  • Each layer transforms the data (weights + bias + activation)
  • Signal moves strictly forward — no loops
  • Final layer produces the output/prediction
  • No learning occurs here — only evaluation

Backpropagation — "Learning from Mistakes"

  • Compares output to the correct answer
  • Measures the error (loss)
  • Error is propagated backward through the network
  • Each neuron adjusts weights & biases to reduce future error
  • This is where learning happens
AspectFeed-ForwardBackpropagation
DirectionInput → OutputOutput error → Input
RoleMake a predictionImprove the prediction mechanism
Learning?NoYes
What flows?Data signalsError signals
When used?Every time the network predictsEvery time the network trains
Analogy: A student solves a problem (feed-forward). The teacher marks it and shows what's wrong (backpropagation). The student updates their understanding to avoid repeating the mistake.
4 · Neural Network Types
CNN

Convolutional Neural Network

Designed for spatial/grid-structured data (images). Think: very smart eyes.

  • Slides small filters over images looking for patterns
  • Early layers: edges/lines → Later: whole objects
  • Uses pooling to reduce size while keeping key info
  • Use cases: image recognition, face detection, medical imaging, autonomous vehicles
RNN

Recurrent Neural Network

Designed for sequential / time-dependent data. Think: a brain reading word by word and remembering.

  • Maintains a hidden state — memory of previous inputs
  • Variants: LSTM (long-term memory via gates), GRU (simpler, faster)
  • Use cases: text, speech, time-series, sensor anomalies
  • Limitation: forgets long-term info; slow for long sequences → replaced by Transformers
Transformer

Transformer

The dominant modern architecture. Think: a powerful reader that looks at ALL words at once.

  • Self-attention: every word evaluates its relationship to every other word
  • Multi-head attention: runs multiple attention processes in parallel
  • Positional encoding: preserves word order (since no recurrence)
  • Encoder: builds global understanding of the full input
  • Decoder: generates output one token at a time, referencing encoder output
  • Scales massively; captures long-range dependencies
  • Powers: ChatGPT, Claude, translation, image generation, multimodal AI
LLM

Large Language Models

Transformer-based models trained on massive text corpora to predict the next token.

  • Work by predicting next token given context
  • Pretrained via self-supervised learning
  • Billions to trillions of parameters
  • Adapted via prompting or fine-tuning (RLHF, instruction tuning)
  • LLMs are a subset of Generative AI (text-focused)
  • Use cases: text gen, QA, translation, code gen, summarization

Why Transformers Dominate

  • Enable autoregressive generation via masked attention
  • Self-attention models long-range dependencies crucial for language
  • Scale extremely well — trillion-parameter models are possible
  • Learn rich probability distributions enabling fluent generation
  • Decoder-only Transformers optimized for next-token prediction (core of GenAI)
  • Unify multimodal generation (text, images, audio, video)
  • Allow emergent behaviors: reasoning, creativity

Transformer Encoder–Decoder in Plain English

  • The encoder reads the entire input and builds a structured mental map of its meaning (each word gets a context-aware representation)
  • The decoder writes the output one word at a time, repeatedly looking back at the encoder's map
  • The decoder uses masked self-attention so it can't "cheat" by seeing future tokens
  • The encoder output never changes during decoding — it acts like a reference document
5 · GANs — Generative Adversarial Networks
What is a GAN?
Two neural networks trained simultaneously in opposition. They learn by competition — generation improves through adversarial feedback.

Generator (G)

  • Takes random noise as input
  • Produces synthetic samples (e.g., images)
  • Goal: fool the discriminator by producing realistic-looking data
  • Improves by learning what the discriminator rejects

Discriminator (D)

  • Receives real data AND generated data
  • Learns to distinguish real from fake
  • Goal: correctly classify inputs as real or generated
  • Acts as a learned critic driving generator improvement
⚙️
Adversarial feedback: the gradient of the discriminator's loss is fed back to the generator. Instead of explicit labels, the generator learns what's wrong by observing how the discriminator rejects its fakes. Training ends when the discriminator can no longer reliably tell real from fake.

GAN Applications

  • Photorealistic face generation (StyleGAN)
  • Artistic rendering & scene generation
  • Super-resolution (SRGAN)
  • Image colorization & style transfer
  • Synthetic training data augmentation
  • Video frame prediction
  • Medical imaging (MRI/CT reconstruction)
  • Text-to-image (before diffusion models)
⚠️
GAN limitation: mode collapse — the generator can get stuck producing only a narrow range of outputs. Training can also be unstable.
6 · Diffusion Models
What is a Diffusion Model?
A probabilistic generative model that learns to generate data by iteratively denoising samples starting from pure noise. It defines a forward process (add noise) and a reverse process (learn to remove that noise).

Forward Process (NN1) — Add Noise

  • Gradually adds small amounts of noise to real data over many steps
  • Eventually turns structured data into pure noise
  • Fixed — not learned
  • Like repeatedly photocopying a photo with blur until it's just static

Reverse Process (NN2) — Remove Noise

  • A neural network trained to remove noise step by step
  • Learns how clean data should look at each stage
  • Usually a U-Net-style architecture
  • This is what's learned during training
🎨
Intuition: start from random noise → clean it a little → clean again → after many steps you have a realistic image. Like sculpting marble — one chip at a time, not drawn in a single stroke.

Key Characteristics

  • Training stability — no adversarial competition (unlike GANs)
  • High sample quality — sharp, diverse, realistic outputs
  • Iterative generation — slower but more controllable
  • Likelihood-based — explicit probability structure
  • Flexible conditioning — guided by text, labels, or reference images
  • Noise schedule — controls how fast noise is added

Diffusion Applications

  • Text-to-image generation, inpainting, super-resolution
  • Music synthesis, speech generation
  • Video generation & frame interpolation
  • Drug & molecule design, medical image reconstruction
  • Denoising and filling missing data
7 · Autoencoders & VAEs

Autoencoder (AE)

Unsupervised neural network that compresses data and reconstructs it — forcing the model to capture the most important features.

  • Encoder — compresses input to a latent representation
  • Latent Space (Bottleneck) — compact feature representation
  • Decoder — reconstructs input from latent representation
  • Deterministic — same input → same encoding
  • Focused on reconstruction, not generation

AE Applications

  • Feature extraction & dimensionality reduction
  • Noise removal
  • Anomaly detection
  • Data compression

Variational Autoencoder (VAE)

A generative model that learns a probabilistic latent space — can generate new, realistic data samples.

  • Encoder — learns a probability distribution for the latent space
  • Latent Space — organized, continuous, smooth sampling
  • Decoder — generates data from sampled latent representations
  • Stochastic — same input can produce different outputs
  • Smooth, structured latent space enables meaningful generation

VAE Applications

  • Data generation & augmentation
  • Anomaly detection with uncertainty
  • Image, audio, and text synthesis
  • Representation learning

  • 1. A neuron uses bias + activation + loss + gradient in a loop to learn. This repeats millions of times.

  • 3. CNN = images (spatial). RNN = sequences (temporal, but limited). Transformer = everything (dominates modern AI).

1 · What Are Intelligent Expert Systems?
Intelligent Expert Systems
Computer systems that solve problems like a human expert — but faster and more accurately. They capture expert knowledge in a structured form (rules) and use inference engines to apply that knowledge to new situations.
🧠
The human brain has three key parts for problem-solving: Memory (knowledge), a Reasoning process (inference), and Interfaces (eyes, ears, skin). Expert systems mirror this exactly.

Examples of Expert System Applications

  • Medical diagnosis and treatment recommendations
  • Industrial equipment fault diagnosis and repair
  • Financial advising
  • Academic advising
  • Psychological/psychiatric assessment
  • Oncology support
  • Recruitment systems (the main example in this module)
2 · Components of an Expert System

Knowledge Base

  • Has two sub-components:
  • Rule Base — created by the developer; contains all the IF-THEN rules (filled at build time)
  • Fact Base — created by the developer but filled at runtime by the user answering questions
  • Equivalent to the brain's memory

Inference Engine

  • The reasoning process — applies the rules to the facts
  • Two types: Forward Chaining (fact-driven) and Backward Chaining (goal-driven)
  • Equivalent to the brain's problem-solving process

Interface

  • How the system interacts with the user — asking questions, receiving answers
  • Equivalent to eyes, ears, skin in the human analogy
3 · Decision Trees
Decision Tree
A diagram that visualizes all the factors in a decision and how one consideration leads to others. Used to map out the problem before converting it to IF-THEN rules. Named because it branches like a tree — at the end of each branch is a conclusion.

Decision Node (Circle)

  • Contains a question / variable
  • Has multiple branches (paths) leaving it
  • Path taken depends on the answer to the question
  • Example: "Does applicant have a degree?" → Yes / No

Conclusion Node (Rectangle)

  • Contains the goal / outcome
  • Signifies a final or sub-conclusion
  • No branches leave it — it's the end of a path
  • Example: "Offer applicant a position in Research"

4 · IF-THEN Rules
📐
Each path through the decision tree from a conclusion node back to the root becomes one IF-THEN rule. The IF part = all decision nodes on that path (connected with AND). The THEN part = the conclusion node.

Rule Generating Technique (Backward from conclusion)

  1. Choose a conclusion (rectangular) node — this becomes your THEN part
  2. Trace backward (left) to find all decision nodes on the path to that conclusion
  3. Continue until no more nodes or another conclusion node is reached
  4. Each decision node becomes one clause in the IF part; connect with AND
  5. The conclusion node becomes the THEN part

5 · Forward Chaining — Fact-Driven
Forward Chaining
Starts with known facts/conditions and works forward through the rules to reach conclusions. Fact-driven: "Given what I know, what can I conclude?"

How Forward Chaining Works (4 Steps)

  1. The system is presented with one or more conditions (facts)
  2. Search the rule base for rules whose IF clause matches a known condition
  3. Each matched rule fires — its THEN part generates new conditions, added to the existing set
  4. Repeat from step 2 with all new conditions until no new conditions can be generated. Session ends.
📈
Stock Market Forward Chaining Example: Known fact: DOLLAR EXCHANGE RATE = FALL
→ Rule 30 fires: INTEREST RATES = RISE
→ Rule 20 fires: STOCK MARKET = FALL
→ No more rules contain STOCK MARKET → session ends
Conclusion: When dollar falls, interest rises, stock market falls.

Forward Chaining Formula (Clause # → Rule #)

Convert clause number to rule number Rule# = [( Quotient(Clause# / 4) + 1 ) × 10]
Example: Clause #1 → Rule# = [(0 + 1) × 10] = 10  |  Clause #5 → Rule# = [(1 + 1) × 10] = 20

Key Data Structures in Forward Chaining

  • Rule Base — all IF-THEN rules
  • Variable List — tracks each variable's value: I (instantiated) or NI (not instantiated)
  • Clause Variable List — maps clause slot numbers to variable names
  • Conclusion Variable Queue — variables whose values are now known and need to be chained forward
  • Clause Variable Pointer — tracks which rule and clause we're currently processing
6 · Backward Chaining — Goal-Driven
Backward Chaining
Starts with a goal/conclusion and works backward through the rules to find what facts are needed to prove it. Goal-driven: "What do I need to know to prove this goal?"

How Backward Chaining Works

  1. User states a goal (e.g., "What position should this applicant get?")
  2. Find the first rule whose THEN clause matches the goal; push it onto the Conclusion Stack
  3. Try to instantiate the IF clause variables one by one (left to right)
  4. If a variable is already known → use it. If it's a conclusion variable → push that rule onto the stack. If neither → ask the user
  5. When all clauses of the top rule are instantiated → execute the rule. If true, the goal is proven. If false, pop the stack and try the next matching rule
  6. Continue until the goal is proven or all rules are exhausted

Backward Chaining Formula (Rule # → Clause #)

Find first clause number for a given rule Clause# = 4 × (Rule# / 10 − 1) + 1
Rule 10 → Clause# = 4×(1−1)+1 = 1  |  Rule 30 → Clause# = 4×(3−1)+1 = 9  |  Rule 40 → Clause# = 4×(4−1)+1 = 13

Key Data Structures in Backward Chaining

  • Knowledge Base — all rules
  • Conclusion Stack — tracks Rule# + Clause# we're currently trying to prove (LIFO)
  • Variable List — tracks each variable: I (instantiated) = known, NI = unknown
  • Clause Variable List — maps clause slot numbers to variable names
  • Conclusion List — which variables appear as THEN (conclusion) variables

Goal: POSITION = ?  |  Inputs discovered: DEGREE=YES, DISCOVERY=NO, GRADE=3.0, EXPERIENCE=4.5

  1. Push Rule 10. DEGREE = NI → ask user: "Degree?" → YES. Rule 10 needs DEGREE=NO → false, pop
  2. Push Rule 30. DEGREE=YES ✓. DISCOVERY=NI → ask user: "Discovery?" → NO. Rule 30 needs DISCOVERY=YES → false, pop
  3. Push Rule 40. QUALIFY=NI but it's a conclusion variable → push Rule 20
  4. Rule 20: DEGREE=YES (known) → fires → QUALIFY=YES. Pop Rule 20
  5. Back to Rule 40: GRADE=NI → ask user: "GPA?" → 3.0. EXPERIENCE=NI → ask: "Years?" → 4.5
  6. Rule 40: QUALIFY=YES ✓, GRADE=3.0<3.5 ✓, EXPERIENCE=4.5≥2 ✓ → POSITION = SERVICE ENGINEER ✓

Forward Chaining — When to Use

  • You have known conditions/facts
  • You want to find all possible consequences
  • The goal is not pre-determined
  • Example: "Given the dollar is falling, what happens?"

Backward Chaining — When to Use

  • You have a specific goal to prove
  • You want to find what facts support it
  • The goal IS pre-determined (select from alternatives)
  • Example: "Is this applicant qualified? For which role?"

1 · What is Machine Learning (Rule Learning)?
Machine Learning / Rule Learning / Knowledge Learning
Instead of having a human expert write rules manually (like in a classic expert system), ML automatically learns those rules from data. Given a dataset of examples and outcomes, the system induces the rules that explain the pattern.
💡
Expert systems require huge human effort to extract rules from domain experts. ML solves this by learning the rules automatically from training data — examples of inputs and correct outputs.

Why Rule Learning Matters

  • Experts are hard to find, expensive, and sometimes unavailable
  • Rules can be extracted from historical decision records
  • The learned rules are human-readable and explainable (unlike black-box models)
  • Works well for structured, tabular decision problems (loans, medical diagnosis, etc.)
2 · Two Types of Learning

Inductive Rule Learning

  • Creates brand new rules about a domain
  • Rules are not derivable from any previous rules
  • Learns from examples — generalizes patterns from data
  • This is what the CPS #4 algorithm does
  • Example: looking at loan records and figuring out what pattern the officer used

Deductive Rule Learning

  • Derives additional rules from already-known domain rules and facts
  • Conclusions could have been reached without the new rules — but faster
  • Enhances efficiency of a system's performance, not its knowledge
  • Like logical inference — deducing implications from existing knowledge
🔑
Key distinction: Inductive = genuinely new knowledge from data. Deductive = faster paths to conclusions already reachable from existing rules.

Important Terminology

  • Training set — the dataset of examples used to learn the rules
  • Positive instance — an example where the outcome = 1 (loan approved)
  • Negative instance — an example where the outcome = 0 (loan rejected)
  • Covers — a rule "covers" an instance if its antecedent (IF part) is True for that instance
  • Specific rule — covers fewer instances (more conditions in antecedent)
  • General rule — covers more instances (fewer conditions)
  • Hypothesis — a learned rule; correct for most inputs but may have errors

4 · The PSR Algorithm
Parameter Selecting Ratio (PSR)
A metric used to pick which parameter to add to a rule next. It answers: "Which parameter, when added, gives me the highest proportion of positive instances among the instances it covers?"
PSR Formula PSR(α) = n⁺_α / n_α
n⁺_α = number of POSITIVE instances covered by parameter α (where α = 1)
n_α = TOTAL instances covered by parameter α (positive + negative)
📐
The higher the PSR, the more "pure" the parameter is — it mostly picks out positive instances. We always pick the parameter with the highest PSR. Ties are broken by choosing the one with the larger sample (more instances).

Method: Separate and Conquer

  1. Start broad: begin with a rule that covers ALL instances (T ⊃ Y)
  2. Compute PSR for every parameter on the current table
  3. Pick the winner — parameter with highest PSR becomes the first condition
  4. Specialize: filter table to only rows where that parameter = 1
  5. Check: any negative instances left? If yes, repeat from step 2 on the filtered table
  6. Rule complete when the current table has no negative instances
  7. Coverage check: does this rule cover ALL positive instances? If not, create a new reduced table (remove positives already covered) and repeat to generate Rule 2, Rule 3, etc.
  8. Done when all positive instances are covered

1 · Game Tree Basics
Adversarial Search (Game Playing)
Competitive environments where agents' goals are in conflict. One player tries to maximize their score, the other tries to minimize it. The AI must plan moves while anticipating the opponent's best responses.

Formal Game Definition — 6 Components

ComponentWhat it means
S₀ — Initial StateHow the game is set up at the start
PLAYER(s)Defines which player has the move in state s
ACTIONS(s)Returns the set of legal moves in state s
RESULT(s, a)The transition model — result of taking action a in state s
TERMINAL-TEST(s)True when the game is over, false otherwise
UTILITY(s, p)Numerical value of the outcome for player p at terminal state s (e.g., +1, 0, −1)

MAX Player

  • Wants to maximize the utility value
  • Moves first
  • High values = good for MAX
  • Represents our AI agent

MIN Player

  • Wants to minimize the utility value
  • Moves second (alternating)
  • Low values = good for MIN
  • Represents the opponent
📐
A ply = one half-move (one player's turn). A "1 move deep" tree has 2 plies. Tic-tac-toe has fewer than 9! = 362,880 terminal nodes. Chess has over 10⁴⁰ — too large to search fully.
2 · Tic-Tac-Toe Programs — 3 Versions
Program #2

Strategy-Based

Uses 4 sub-functions with intelligent strategy. Board values: 2=blank, 3=X, 5=O.

  • Make1 — returns a corner square (1,3,7,9)
  • Make2 — returns center (5) if blank, else a non-corner square (2,4,6,8)
  • Posswin(p) — returns 0 if player p can't win next move; otherwise returns the winning square number
  • Go(n) — makes a move in square n
  • Posswin works by multiplying row values: product 18 (3×3×2) = X can win; product 50 (5×5×2) = O can win
Program #3

Magic Square Method

Replaces row/column scanning with arithmetic. Uses a magic square layout where all rows, columns, diagonals sum to 15.

  • Board positions assigned so any 3-in-a-row sums to 15
  • To check for a win: look at each pair of your squares, compute 15 − (sq1 + sq2)
  • If the result is 1–9 and that square is blank → winning move
  • If out of range (≤0 or >9) or occupied → skip
  • Much more efficient than checking all rows/columns

  • X checks pair (5,6): 15−(5+6)=4 → square 4 is blank → X moves to 4 to block O's win!
4 · Minimax Algorithm
Minimax
Computes the optimal move by recursively exploring the full game tree. At MAX nodes: choose the move with the highest backed-up value. At MIN nodes: choose the move with the lowest backed-up value. Values propagate back up from terminal nodes.

Minimax Algorithm Steps

  1. If DEEP-ENOUGH (terminal or depth limit reached) → return EVALUATION-FUNCTION(position, player)
  2. Generate successors using MOVE-GEN(position, player)
  3. If no successors → treat same as step 1
  4. For each successor: recursively call MINIMAX with OPPOSITE(player) and depth+1
  5. Negate the returned value (negamax style: NEW-VALUE = −VALUE(result))
  6. Track best score and best path; return when all successors examined
💡
Why negate? The algorithm uses negamax — from any player's perspective, the opponent's gain is your loss. Negating lets one function handle both MAX and MIN.

DEEP-ENOUGH Checks

  • Has one side won?
  • How many plies already explored?
  • How promising is this path?
  • How much time is left?
  • How stable is the configuration?

MOVEGEN Output

  • Returns list of child nodes (successors)
  • Represents all legal moves from current position
  • If empty → no moves, treat as terminal

Minimax Worked Example (from slides)

Tree: Root A (MAX) → B, C, D (MIN) → leaf values

  • B (MIN) → children with values: MIN picks lowest → backs up 3
  • C (MIN) → children → backs up 2
  • D (MIN) → children → backs up 2
  • A (MAX) → MAX(3, 2, 2) = 3 → Best path: A → C → I (Best Score: −2 from MIN's perspective)
5 · Evaluation Functions (Imperfect Real-Time Decisions)
⚠️
When the tree is too deep to reach terminal nodes in time, cut off the search at depth d and apply an evaluation function to the cutoff nodes — treating them as if they were terminal leaves.

3 Requirements for a Good Evaluation Function

  1. Correct ordering: must order terminal states the same way as the true utility function — wins better than draws, draws better than losses
  2. Fast computation: the whole point is to search faster — evaluation can't be slow
  3. Strong correlation: for non-terminal states, must be strongly correlated with actual winning chances
Weighted Linear Evaluation Function EVAL(s) = w₁·f₁ + w₂·f₂ + … + wₙ·fₙ
wᵢ = weight of feature i (e.g., piece value) · fᵢ = value of feature i in the state (e.g., number of queens)
♟️
Chess example: Pawn=1, Knight/Bishop=3, Rook=5, Queen=9. Add up all your piece values minus opponent's = material value. Other features like "king safety" add fractions of a pawn.

Tic-Tac-Toe Heuristic (from slides)

  • Count all winning lines open to MAX → subtract all winning lines open to MIN
  • Forced win for MAX = +∞
  • Forced win for MIN = −∞
  • Maximize this difference at each search step
6 · Alpha-Beta Pruning
Alpha-Beta Pruning
An optimization of Minimax that prunes (ignores) branches that cannot possibly affect the final decision. Explores depth-first. Uses two thresholds: Pass-Thresh (best found so far for current player) and Use-Thresh (best the opponent can guarantee). Prune when Pass-Thresh ≥ Use-Thresh.

Pass-Thresh (α equivalent)

  • The best value the current player has found so far
  • Can only increase (we keep finding better moves)
  • Initially set to the minimum possible evaluation value
  • Updated whenever a better successor is found

Use-Thresh (β equivalent)

  • The best value the opponent is guaranteed from a higher level
  • Can only decrease (passed in from parent)
  • Represents "the opponent won't let you do better than this"
  • When you pass it to the recursive call, you negate both thresholds
✂️
Pruning condition: If Pass-Thresh ≥ Use-Thresh → stop exploring this branch. The opponent at the level above would never allow this path because we've already found something better.

Alpha-Beta Algorithm Key Steps

  1. Call: MINIMAX-A-B(position, depth, player, Use-Thresh, Pass-Thresh)
  2. For each successor, recurse with swapped & negated thresholds: (−Pass-Thresh, −Use-Thresh)
  3. Negate returned value: NEW-VALUE = −VALUE(result)
  4. If NEW-VALUE > Pass-Thresh → update Pass-Thresh and best path
  5. If Pass-Thresh ≥ Use-Thresh → PRUNE (return immediately)
  6. After all successors: return Pass-Thresh and best path

Alpha-Beta Worked Example (from slides)

Same tree as Minimax example. Terms: CPT=Change Pass Threshold, NCPT=No Change, NPr=No Pruning, Pr=Prune

  • Alpha-Beta explores fewer nodes than Minimax by pruning branches
  • Best path: A → C → H
  • Best score at root: 4
  • Result may differ from plain Minimax because pruning can affect which path is selected
💡
Terminal node in MIN's turn: If a terminal node is encountered when it's MIN's turn, you must reverse (negate) the f-value before using it in the algorithm.
